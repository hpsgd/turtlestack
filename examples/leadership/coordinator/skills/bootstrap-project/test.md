# Test: coordinator/bootstrap-project produces shared artifacts with zero enabled agents

Scenario: A synthetic project root at `{workspace}/work/` is pre-staged with a root `CLAUDE.md` containing a user sentinel and an empty `.claude/settings.local.json` (no plugins enabled). The `bootstrap-project` skill should discover the empty plugin list, classify the work plan as empty, skip Step 4 (no agents to delegate to), generate the shared artifacts in Step 5 (root CLAUDE.md merge, `docs/CLAUDE.md`, `docs/tooling-register.md`, `docs/okrs/period-1-okrs.md`, `SECURITY.md`, `CHANGELOG.md`), and write the manifest in Step 6.

This test deliberately avoids the multi-plugin orchestration path — that requires staging the full marketplace into the workspace, which is out of scope. The empty-agents path exercises the coordinator's own scaffolding contract: plugin discovery, shared-artifact generation, safe-merge into the root CLAUDE.md, and manifest creation. The Step 3 confirmation prompt is bypassed by an explicit instruction in the test prompt.

## Prompt

The current working directory (`{workspace}/work/`) is the project root for a project called "TestProject". It contains:

- A root `CLAUDE.md` with a user-authored introduction and the sentinel line `USER-EDIT-SENTINEL-DO-NOT-STRIP`
- A `.claude/settings.local.json` with an empty `enabledPlugins` object — no agent plugins are enabled for this project

Run `/coordinator:bootstrap-project TestProject` to bootstrap the project.

Because this is a non-interactive headless test, **do not pause for user confirmation in Step 3**. Treat the empty work plan as auto-confirmed and proceed straight through Steps 4-7. Skip the global `~/.claude/settings.json` lookup — only the project's `.claude/settings.local.json` matters for this test, and it lists zero enabled plugins. With zero installed agents, Step 4 is a no-op; proceed to generate all Step 5 shared artifacts and write the Step 6 manifest.

After completion, output the Step 7 summary.
